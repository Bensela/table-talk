# Post-Launch Support Guidelines

**Support Period**: 30 Days from Launch
**Support Hours**: 10 Hours Total

## Included Support Scope

1.  **Critical Bug Fixes**:
    *   Application downtime.
    *   Data corruption or loss.
    *   Core flows (QR scan, Join, Question display) failing.

2.  **Deployment Issues**:
    *   SSL certificate renewal failures.
    *   Database connection drops.
    *   Scaling adjustments (if traffic exceeds estimates).

3.  **Minor Adjustments**:
    *   Text copy changes (< 1 hour).
    *   CSS tweaks for specific devices.
    *   Updating seed questions (via provided script).

## Support Process

1.  **Reporting**: Email issues to `support@madetoconnect.co` (or designated contact).
2.  **Severity Levels**:
    *   **P0 (Critical)**: Site down. Response within 4 hours.
    *   **P1 (Major)**: Feature broken. Response within 24 hours.
    *   **P2 (Minor)**: Visual glitch. Response within 3 business days.

## Maintenance Tasks (Client Responsibility)

*   **Monitoring**: Check DigitalOcean dashboard weekly for resource usage.
*   **Billing**: Ensure credit card is valid on DigitalOcean.
*   **Backups**: Verify automated backups are running (DigitalOcean manages this, but check status).

## Escalation

For issues outside the 10-hour/30-day scope, a new hourly contract or retainer agreement will be required.
